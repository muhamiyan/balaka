# Release 2026.02.4-RELEASE

## Highlights

- **Migration Consolidation**: 11 Flyway migrations (V001-V011) consolidated into 4 domain-based files (V001-V004) for cleaner structure and easier maintenance.

## What's New

### Migration Consolidation

Production migrations restructured from 11 scattered files into 4 coherent, domain-based files:

| File | Contents |
|------|----------|
| V001__security.sql | users, user_roles, audit_logs + default admin seed |
| V002__core_schema.sql | COA, templates, transactions, clients, projects, invoices, vendors, bills, employees, payroll, assets, inventory, documents, tax |
| V003__feature_schema.sql | Device auth, bank recon, analysis reports, transaction tags, smart alerts, payment tracking, recurring transactions |
| V004__seed_data.sql | Transaction sequences, bank parser configs, alert rules |

Previously scattered elements are now properly grouped:
- V005 `ALTER TABLE` additions (API fields, AI metadata) are inlined into V002 `CREATE TABLE` statements
- V007 `analysis_reports` table (misplaced in bank recon seed file) moved to V003
- V009 alert seed data separated from schema into V004

## Breaking Changes

**Database migration compatibility**: This release changes the Flyway migration file names and checksums. Existing deployments will fail to start without manual intervention.

## Migration Guide

**IMPORTANT: Existing deployments require a manual `flyway_schema_history` reset before upgrading.**

The database schema itself is unchanged — only the Flyway tracking metadata needs to be updated. No data is lost.

### Steps

1. Stop the application:
   ```bash
   sudo systemctl stop aplikasi-akunting
   ```

2. Reset `flyway_schema_history`:
   ```sql
   BEGIN;
   DELETE FROM flyway_schema_history;
   INSERT INTO flyway_schema_history
     (installed_rank, version, description, type, script, checksum, installed_by, installed_on, execution_time, success)
   VALUES
     (1, '001', 'security', 'SQL', 'V001__security.sql', 998030233, 'akunting', NOW(), 0, TRUE),
     (2, '002', 'core schema', 'SQL', 'V002__core_schema.sql', 1796101559, 'akunting', NOW(), 0, TRUE),
     (3, '003', 'feature schema', 'SQL', 'V003__feature_schema.sql', 2095885774, 'akunting', NOW(), 0, TRUE),
     (4, '004', 'seed data', 'SQL', 'V004__seed_data.sql', -1590895538, 'akunting', NOW(), 0, TRUE);
   COMMIT;
   ```

3. Deploy the new JAR and start:
   ```bash
   sudo systemctl start aplikasi-akunting
   ```

4. Verify the application starts without Flyway errors.

### Fresh installations

No special steps needed. Flyway will run V001-V004 from scratch.
